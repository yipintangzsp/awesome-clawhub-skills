## ⏰ Cron 配置

### 发布提醒 (飞书)
```bash
# 每天 19:00 发送次日发布计划
0 19 * * * /path/to/send-feishu-reminder.sh --next-day

# 发布前 1 小时紧急提醒
0 11,19 * * * /path/to/send-feishu-reminder.sh --urgent

# 每周一 09:00 发送周计划
0 9 * * 1 /path/to/send-feishu-reminder.sh --weekly
```

```bash
# 每天 10:00 提醒更新昨日数据
0 10 * * * /path/to/send-review-reminder.sh --daily

# 每周一 10:00 提醒更新周数据
0 10 * * 1 /path/to/send-review-reminder.sh --weekly
```

## 🛠️ 自动化脚本

### 1. 发送飞书提醒
```bash
#!/bin/bash
# send-feishu-reminder.sh

FEISHU_WEBHOOK="https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
DATE=$(date -d "+1 day" +%Y-%m-%d)

# 获取次日发布计划
PLAN=$(grep "$DATE" pending-posts/发布提醒配置.md | grep -A5 "### $DATE")

# 发送飞书消息
curl -X POST "$FEISHU_WEBHOOK" \
  -H "Content-Type: application/json" \
  -d "{
    \"msg_type\": \"interactive\",
    \"card\": {
      \"header\": {
        \"title\": {
          \"tag\": \"plain_text\",
          \"content\": \"📢 明日发布提醒\"
        }
      },
      \"elements\": [
        {
          \"tag\": \"markdown\",
          \"content\": \"$PLAN\"
        }
      ]
    }
  }"
```

### 2. 数据复盘提醒
```bash
#!/bin/bash
# send-review-reminder.sh

TYPE=$1  # --daily or --weekly

if [ "$TYPE" == "--daily" ]; then
  REMINDER="请更新昨日发布文章的数据：曝光量、点赞、评论、转化数"
elif [ "$TYPE" == "--weekly" ]; then
  REMINDER="请进行周报复盘：哪个平台效果最好？如何优化？"
fi

# 发送飞书消息
curl -X POST "$FEISHU_WEBHOOK" \
  -H "Content-Type: application/json" \
  -d "{
    \"msg_type\": \"text\",
    \"content\": {
      \"text\": \"📊 数据复盘提醒\n\n$REMINDER\n\n追踪表：pending-posts/发布追踪表.csv\"
    }
  }"
```

## 📞 故障排除

### 问题：提醒未发送
- 检查 cron 服务是否运行：`systemctl status cron`
- 检查脚本权限：`chmod +x *.sh`
- 检查飞书 webhook 是否有效

### 问题：文章计数错误
- 手动运行：`find pending-posts/ -name "*.md" | wc -l`
- 检查是否有隐藏文件

### 问题：时区错误
- 确认服务器时区：`timedatectl`
- 设置正确时区：`timedatectl set-timezone Asia/Shanghai`
