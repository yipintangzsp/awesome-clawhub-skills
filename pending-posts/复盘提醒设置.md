## 🎯 复盘周期

### 24 小时快速复盘
**目的**: 快速判断内容表现，及时调整策略  
**时间**: 发布后 24 小时  
**检查项**:
- 曝光量是否达到预期？
- 互动率 (点赞 + 评论)/曝光 是否 >1%？
- 是否有引流私信/点击？
- 是否需要二次推广？

### 7 天深度复盘
**目的**: 全面评估内容价值，优化后续策略  
**时间**: 发布后 7 天  
**检查项**:
- 总曝光量趋势
- 互动质量分析
- 引流转化效果
- 收入增长贡献
- 可复制的经验

### 30 天月度复盘
**目的**: 平台策略优化，内容方向调整  
**时间**: 每月 1 日  
**检查项**:
- 各平台 ROI 对比
- 最佳内容类型
- 最佳发布时间
- 下月发布计划调整

## 🛠️ 自动化脚本

### 1. 24 小时复盘提醒脚本
```bash
#!/bin/bash
# 24h-review-reminder.sh

FEISHU_WEBHOOK="https://open.feishu.cn/open-apis/bot/v2/hook/xxx"
TRACKING_FILE="pending-posts/发布追踪表 - 完整版.csv"

# 获取 24 小时前发布的文章
YESTERDAY=$(date -d "-1 day" +%Y-%m-%d)
ARTICLES=$(grep "$YESTERDAY" "$TRACKING_FILE" | grep -v "待发布")

while IFS=, read -r platform file title publish_time rest; do
  if [ -n "$publish_time" ] && [ "$publish_time" != "待发布" ]; then
    # 发送飞书提醒
    curl -X POST "$FEISHU_WEBHOOK" \
      -H "Content-Type: application/json" \
      -d "{
        \"msg_type\": \"interactive\",
        \"card\": {
          \"header\": {
            \"title\": {
              \"tag\": \"plain_text\",
              \"content\": \"📊 24 小时复盘提醒\"
            }
          },
          \"elements\": [
            {
              \"tag\": \"markdown\",
              \"content\": \"文章：$title\n平台：$platform\n请更新 24 小时数据！\n追踪表：$TRACKING_FILE\"
            }
          ]
        }
      }"
  fi
done <<< "$ARTICLES"
```

## 📊 数据看板

### 关键指标定义
| 指标 | 计算公式 | 目标值 |
|------|----------|--------|
| 曝光量 | 平台统计 | 单篇>1000 |
| 互动率 | (点赞 + 评论)/曝光 | >1% |
| 引流率 | 私信数/曝光 | >0.1% |
| 转化率 | 转化数/私信数 | >10% |
| ROI | 收入/时间成本 | >200% |

### 平台基准线
| 平台 | 平均曝光 | 平均互动率 | 平均转化率 |

## ⚠️ 注意事项

1. **数据准确性**: 确保从平台官方 analytics 获取数据
2. **时间一致性**: 每天固定时间记录数据 (建议 10:00)
3. **异常处理**: 发现数据异常及时标注并分析原因
4. **隐私保护**: 用户私信内容不要公开分享
5. **持续优化**: 根据复盘结果调整发布策略
