**技术实现**：
```javascript
// 核心逻辑
async function scanMemeToken(contractAddress, chain) {
  // 1. 获取合约信息
  const contract = await getContract(contractAddress, chain);
  
  // 2. 检查 LP 锁定
  const lpInfo = await checkLPLock(contractAddress);
  const lpLocked = lpInfo.locked && lpInfo.lockDuration > 30;
  
  // 3. 检查合约权限
  const permissions = await analyzePermissions(contract);
  const canMint = permissions.ownerCanMint;
  const canChangeTax = permissions.ownerCanChangeTax;
  
  // 4. 检查持币分布
  const holders = await getTopHolders(contractAddress, 10);
  const top10Percent = holders.reduce((sum, h) => sum + h.percent, 0);
  
  // 5. 计算风险分数
  let riskScore = 0;
  if (!lpLocked) riskScore += 30;
  if (canMint) riskScore += 25;
  if (canChangeTax) riskScore += 20;
  if (top10Percent > 50) riskScore += 25;
  
  return {
    riskScore,
    level: riskScore < 30 ? '安全' : riskScore < 60 ? '中等' : '高危',
    details: { lpInfo, permissions, holders }
  };
}
```

## 七、避坑指南：我踩过的 3 个大坑

### 坑 1：过度承诺

最初我写"100% 检测 Rug Pull"，结果有用户买了被 Rug 的币，来找我退款。

**教训**：
- 不要承诺 100% 准确
- 明确说明工具局限性
- 加上免责声明

**现在的免责声明**：
```
⚠️ 本工具仅供参考，不构成投资建议。
链上交易风险极高，请自行研究（DYOR）。
```

### 坑 2：忽视 Gas 费

最初我每次扫描都调用链上 RPC，结果 Gas 费比收入还高。

**教训**：
- 优先使用 API（比 RPC 便宜）
- 缓存扫描结果（避免重复查询）
- 批量处理请求（降低单次成本）

**优化后**：
- 单次扫描成本：¥0.05 → ¥0.01
- 利润率：85% → 95%

### 坑 3：单打独斗

最初所有事情自己干，后来加入社区才发现：
- 有人可以合作开发
- 有人可以互相推广
- 有人可以分享资源

**教训**：加入社区，抱团取暖。
