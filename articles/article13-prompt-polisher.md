## 🤖 为什么 AI 总说废话？

你是不是也这样：

```
你："帮我写个文案"
AI："好的，请问您需要什么类型的文案？..."

你："分析一下这个市场"
AI："当然，为了给您更准确的分析..."

你："做个策划方案"
AI："我很乐意帮助您..."
```

**AI 说了很多，但什么都没说。**

问题不在 AI，在你的 Prompt 太烂。

---

## 🎯 好 Prompt vs 烂 Prompt

### 烂 Prompt
```
"帮我写个文案"
```

**问题：**
- ❌ 没有人设
- ❌ 没有约束
- ❌ 没有格式
- ❌ 质量不可控

**输出：** 泛泛而谈，没法用

---

### 好 Prompt
```
# Role
你是一名资深文案专家，拥有 10 年广告行业经验

# Task
为一款减肥产品写小红书文案

# Constraints
- 标题必须有情绪词和 Emoji
- 正文必须包含 3 个用户痛点
- 必须有行动号召
- 字数 300-500 字

# Output Format
1. 标题（5 个选项）
2. 正文
3. 热门标签（5 个）

# Quality Standards
- 专业准确
- 逻辑清晰
- 实用可执行
```

**输出：** 直接能用，不用改

## 📝 Prompt 优化框架（CRAFT 模型）

### 1️⃣ C - Context（背景）

**给 AI 提供背景信息。**

```
❌ 没有人设
✅ "你是一名资深文案专家，拥有 10 年广告行业经验"
✅ "你是一名麦肯锡高级合伙人"
✅ "你是一名哈佛招生官"
```

**为什么有效？**
- AI 会模仿该角色的思维方式
- 输出更专业
- 语气更准确

---

### 2️⃣ R - Role（角色）

**明确 AI 的身份和职责。**

```
# Role
- 你是 XX 领域的专家
- 你有 XX 年的经验
- 你擅长 XX 技能
```

**案例：**
```
# Role
你是一名资深跨境电商选品顾问，服务过 100+ 卖家，
帮助他们从 0 做到日出百单。
```

---

### 3️⃣ A - Action（行动）

**明确告诉 AI 要做什么。**

```
❌ "帮我写个文案"
✅ "为一款减肥产品写小红书文案，目标用户是 25-35 岁女性"
```

**为什么有效？**
- 减少歧义
- AI 知道具体要什么
- 输出更精准

---

### 4️⃣ F - Format（格式）

**告诉 AI 怎么输出。**

```
# Output Format
1. 【问题分析】- 理解用户需求
2. 【核心思路】- 解决框架
3. 【详细方案】- 分步骤执行
4. 【注意事项】- 风险提示
5. 【延伸建议】- 额外价值
```

**为什么有效？**
- 输出结构化
- 易于阅读
- 方便使用

---

### 5️⃣ T - Tone（语气）

**设定输出的语气和风格。**

```
# Tone
- 专业但不生硬
- 直接但不粗鲁
- 实用可执行
- 避免 AI 套话（"作为 AI 语言模型..."）
```

### 案例 1：文案写作

**优化前：**
```

```


---

### 案例 2：市场分析

**优化前：**
```
"分析一下这个市场"
```

**优化后：**
```
# Role
你是一名麦肯锡高级合伙人

# Task
分析中国宠物用品市场

# Constraints
- 必须用数据支撑观点
- 必须分析竞争格局
- 必须给出进入建议

1. 市场规模
2. 竞争格局
3. 用户画像
4. 进入建议
5. 风险提示

# Quality Standards
- 数据注明来源或估算依据
- 建议具体可执行
```

**效果：** 报告级别输出，可直接用

## 💡 常用 Prompt 模板

### 模板 1：内容创作
```
# Role
你是一名资深 [领域] 专家

# Task
为 [目标受众] 创作 [内容类型]

# Constraints
- [约束 1]
- [约束 2]
- [约束 3]

# Output Format
1. [部分 1]
2. [部分 2]
3. [部分 3]
```

### 模板 2：问题分析
```
# Role
你是一名 [领域] 顾问

# Task
分析 [问题]

# Constraints
- 必须用数据支撑
- 必须给出解决方案
- 必须评估风险

# Output Format
1. 问题分析
2. 解决方案
3. 风险评估
4. 行动建议
```
